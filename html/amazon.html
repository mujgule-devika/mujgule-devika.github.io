The Problem

Many large businesses are reacting to events in Ukraine finding ways to promote accurate information, and support the people of Ukraine. Amazon doesn’t have a robust feature set that does this, so let’s explore, and design a solution. Furthermore, let’s design this solution (or set of solutions) in such a way that it can be repuroposed for future humanitarian crisises, or major social issues.

Approach to Design Solution: Design Sprint

We approached the problem as a Team of 4 and structured the Design Sprint and worked as a group to answer few of the following questions

  • What are the questions we’ll need to answer in this sprint?
  • Discussion: who do we think will use this product?
  • Discussion: imagine the perfect outcome for this project, what would it do for customers, for businesses, for Amazon?
  • How might we: Anything that sounds difficult, or feels like an opportunity capture as a how might we.
  • Rapid competitve research
  • Explore the Amazon UI
  • Discussion: imagine this project was a total failure - what could go wrong? Why would customers hate it, why would Amazon remove this feature?

Initial Sketches & Dot Voting

After the Sprint and answering the questions we short listed a set of features and made initial hand sketches.

A few set of features that we brainstromed were:

  • A different screen for Ukraine (or any social cause)
  • Entry points: Support Ukraine Tab, Dedicated Card on home screen
  • Educational Material: Prime Video recommendations, Amazon Music, Podcasts
  • Hashtag feature to highlight products and in a search bar
  • Donations at Checkout

Hand Sketches

Hand Sketches are a very powerful tool - we leveraged this even more to explore solutions - explore more screens in more detail. The main objective was to create designs quickly at a low cost and in much greater detail. We skipped wireframing in this project and heavliy leveraged Hand Sketches. This was easy, quick and low-effort. It helped my brainstrom multiple design solutions for a solution in mind.

High Fidelity Prototypes

The design aims at clear and intuitive feature set. A roboust amount of features to support the cause. It deliveres multiple features and ways to aid a social cause. These set of features are designed in a manner that they can be repurposed and reused for different social causes.

Feature set and Prototype

  • Dedicated entry menu point, card and new page for the cause
  • Redesigned the existing donation page with additional options to donate to
  • Educational Material: News & Podcasts
  • Dedicated page for all sellers from Ukraine and supporting Ukriane
  • Hashtag to categorise and search products quickly
  • Donation at Check-out with option to select organization to donate to

Ways to Repurpose and Reuse Features

  • Can be replace with different causes. A new dedicated screen is added which can be repurposed
  • Educational Material: This will source information regarding furure cause
  • Can be used again to show support by sellers or source different products e.g sustainable products, etc along with hashtage feature
  • Donation at Check-out with option to select organization to donate can be replaced with other source to donate to

Running Usability Tests

The users were asked to share what they were doing, thinking, and feeling while interacting with the prototype and trying to complete the tasks given to them. I tested around the key tasks I identified earlier in the process, asking the user to browse the overall app, select distracting apps, go to stats page, set notification message, set timer for app and blacklist some apps.

Usability Test Results & Iteration

User testing helped me uderstand the drawbacks in my app and the changes I need to make for the same

Insights

  • Message text on product was too lengthy
  • Text on entry point card on cause page was not intuitive for user
  • Needed option on checkout donation page to select which cause to donate to
  • Fonts on some product cards were small
  • Some tag and labels seem redundant

Iteration and Changes

  • Shortened text and create another screen for details
  • Changed text to make it more intuitive
  • Added radio button to select which cause to donate to at checkout
  • Changed fonts on product cards
  • Generalized hashtag and separated labels to make more sense

Conclusion

What went well?

  • UI replicated Amazon look and feel.
  • Features were intuitive and easy to notice
  • Designed a more robust feature set to support humanitarian cause
  • Interview process help me understand what exactly user need vs what I think they need
  • Could create simple designs and remove complicated design patterns

What could be done in the future?

  • Amazon Prime videos could be integrated to provide live stream news, documentaries, etc for educational material
  • Brand wise products could be added that support humanitarian causes
  • Recommendations on similar products that support a cause based on search/purchase history of user
