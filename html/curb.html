UX Research

Understanding user pain points

User Interviews

With the screen time manager app I want to learn if users think it’s important to manage and track their screen time for wellbeing and what apps they’re using for it. I am trying to understand what kind of goals users want to achieve from the app. Like better productivity, overcome obsession with particular apps or phone, restrict usage, etc.. I want to understand how effective the apps have been in helping them reduce their screen time. I would like to know what works and doesn’t work in the app they use.

Wireframes

After finalizing the best options from hand sketches I worked on making wireframes. Before working on the visual design, I wanted to first make sure that the design was functional. In order to do this, I decided to create a mid-fidelity prototype which would help me quickly test the design and make any revisions before integrating the branding and visual design. I created mid-fidelity wireframes on Figma of the key screens the users would be interacting with.

User Testing

Test Preparation

For usability testing, I conducted interviews with 6 people. I explained the motive and goal of the app. Prepared list of questions to ask the used before and after running the usability tests and prepared a set of tasks for them to carry out.

Running Usability Tests

The users were asked to share what they were doing, thinking, and feeling while interacting with the prototype and trying to complete the tasks given to them. I tested around the key tasks I identified earlier in the process, asking the user to browse the overall app, select distracting apps, go to stats page, set notification message, set timer for app and blacklist some apps.

Usability Test Results

User testing helped me understand the drawbacks in my app and the changes I need to make for the same

Insights

  • Need less overwhelming cooler
  • Drag and drop to blacklist is not intuitive
  • Need less effort to navigate app

Changes

  • Changed color palette
  • Used different design to add or remove apps from blocklist and timer feature
  • Added better interactions

Conclusion & Reflections

What went well?

  • Clean UI and easier workflow
  • Solved user pain problems to certain extent
  • Interview process help me understand what exactly user need vs what I think they need
  • Could create simple designs and remove complicated design patterns

What could be done in the future?

  • Create solutions for all the pain points, which given more time can be achieved
  • Goals and focus mode feature can be integrated

  • Weekly automated dashboards, widgets, etc can be added
